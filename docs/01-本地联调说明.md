# 本地联调说明

这个项目已经收敛成纯本地联调版，目标是让新同学 clone 后尽快跑起来。

## 环境要求

- Docker Desktop 或 Docker Engine + Docker Compose
- 微信开发者工具

## 启动步骤

在项目根目录执行：

```bash
chmod +x scripts/*.sh
./scripts/dev-up.sh
```

查看状态：

```bash
./scripts/dev-status.sh
```

停止环境：

```bash
./scripts/dev-down.sh
```

如需清空旧数据并重新初始化：

```bash
docker compose down -v
./scripts/dev-up.sh
```

## 启动后地址

- 管理后台：`http://127.0.0.1:5173`
- 后端 Swagger：`http://127.0.0.1:8080/swagger-ui.html`
- 健康检查：`http://127.0.0.1:8080/actuator/health`

## 小程序联调

`./scripts/dev-up.sh` 会自动生成 `miniapp/config.js`。

如果微信开发者工具运行在 Windows、后端运行在 WSL，可以额外执行：

```bash
./scripts/configure-miniapp-api.sh
```

然后用微信开发者工具导入 `miniapp/`，使用默认测试账号登录即可。

## 常见问题

### 后台或接口访问失败

先执行 `./scripts/dev-status.sh`，确认 `mysql`、`redis`、`backend`、`admin` 四个容器都在运行。

### 小程序提示网络不可达

检查 `miniapp/config.js` 中的接口地址是否可从开发者工具访问；必要时重新运行 `./scripts/configure-miniapp-api.sh`。
