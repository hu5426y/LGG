# 真机调试说明

## 目标

让手机微信能够访问你本机运行的后端接口，避免登录点击后没有响应。

## 推荐方式

优先使用临时公网 HTTPS 隧道，把本地 `http://127.0.0.1:8080` 暴露成一个手机可访问的 HTTPS 地址。

常见可选工具：

- `cloudflared tunnel --url http://127.0.0.1:8080`
- `ngrok http 8080`

拿到公网地址后，用仓库脚本写入小程序配置：

```bash
cd ~/dev/LJJ
./scripts/configure-miniapp-api.sh https://your-tunnel-domain.example.com
```

脚本会自动补齐 `/api`，最终写入 `miniapp/config.js`。

## 真机联调步骤

1. 启动本地后端，确认 `http://127.0.0.1:8080/actuator/health` 可用。
2. 启动临时 HTTPS 隧道，并记下公网地址。
3. 运行 `./scripts/configure-miniapp-api.sh <你的 HTTPS 地址>`。
4. 在微信开发者工具中重新编译小程序。
5. 真机预览或真机调试前，确认小程序后台已配置合法 `request` 域名；如果只是开发联调，也可以在开发设置里临时关闭域名校验。

## 排查要点

- 如果登录页点击无响应，先看登录页展示的“当前接口”是不是 HTTPS 公网地址。
- 如果是真机专有问题，但开发者工具正常，优先怀疑当前接口地址对手机不可达。
- 如果接口变更后仍旧连旧地址，重新执行一次 `./scripts/configure-miniapp-api.sh`，然后重新编译小程序。
