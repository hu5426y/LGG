# 校园乐跑项目任务清单

## 1. 交付目标

- 完成微信小程序学生端、Spring Boot 后端、网页管理端三端联动项目。
- 项目可本地启动、可演示、可用于毕业设计答辩展示。
- 交付完整文档，包括任务清单、开发文档、部署说明、开发日志。

## 2. 必做范围

- 学生端
  - 登录与个人中心
  - 首页轮播与推荐信息
  - 跑步打卡：开始、暂停、结束、轨迹点上传、配速与里程展示
  - 成就与任务系统
  - 排行榜
  - 跑步动态发布、点赞、评论
  - 活动列表与报名
- 后端
  - JWT 鉴权与角色权限
  - 用户、跑步记录、成就任务、动态互动、活动报名、排行榜、日志审计
  - MySQL 数据持久化
  - Redis 排行榜缓存
  - Swagger/OpenAPI 接口文档
  - 初始化演示数据
- 管理端
  - 登录
  - 数据概览
  - 学生列表与详情
  - 动态审核与热门推荐干预
  - 活动发布与教程管理
  - 勋章等级维护
  - 审计日志查看

## 3. 开发阶段

- 阶段 A：需求拆解与方案落地
  - 输出系统架构、模块边界、数据库设计、接口分层方案
- 阶段 B：基础工程搭建
  - 创建 monorepo 目录
  - 创建后端、管理端、小程序端工程
  - 创建 Docker Compose 基础环境
- 阶段 C：后端核心功能
  - 完成认证、用户、跑步、动态、活动、排行榜、管理接口
  - 完成数据库脚本与演示种子数据
- 阶段 D：前端实现
  - 小程序端核心页面与接口接入
  - 管理端核心页面与接口接入
- 阶段 E：交付收尾
  - 构建验证
  - README、部署说明、答辩演示说明
  - 更新开发日志

## 4. 验收标准

- 后端可通过 `mvn test` 与 `mvn package`。
- 管理端可通过 `npm run build`。
- 小程序端目录结构完整，具备微信开发者工具导入条件。
- Docker Compose 可拉起 MySQL 与 Redis。
- README 能指导评审完成本地部署。
- 开发日志记录当前完成度、下一步任务、风险点。
