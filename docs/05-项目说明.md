# 校园乐跑项目说明

## 1. 项目概述

本系统用于高校学生跑步打卡、成长激励与运营管理，采用“小程序端 + 后端 + 管理端”的三端协同模式，当前交付目标是单校内部真实上线 MVP。

## 2. 已实现模块

### 2.1 学生端

- 微信登录与学号绑定
- 首页轮播、排行榜摘要、任务进度、活动推荐
- 真实 GPS 跑步：开始、暂停、继续、结束
- 跑步结果上传与历史记录查询
- 动态发布、点赞、评论、举报
- 活动报名与教程查看
- 个人中心、勋章、任务进度

### 2.2 后端

- JWT 登录鉴权
- 微信登录适配与绑定流程
- 用户信息与学生名册管理
- 跑步记录管理与有效性校验
- 游戏化任务与勋章发放
- 排行榜缓存
- 动态内容审核
- 活动与教程管理
- 审计日志与健康检查

### 2.3 管理端

- 控制台总览
- 学生名单导入
- 学生列表、状态启停用、微信绑定重置
- 内容审核与推荐干预
- 活动发布
- 跑步教程管理
- 勋章维护
- 操作日志查询

## 3. 首发版本边界

- 后端采用模块化单体，优先保证单校部署与运维复杂度可控
- 学生端以微信登录为主，不开放学生自注册
- 学生名单由后台预导入，不在首发范围内做复杂组织同步
- 跑步默认采用真实 GPS 轨迹，模拟模式仅保留给开发环境

## 4. 后续可扩展方向

- 接入更精细的轨迹风控与作弊识别
- 增加学院/班级分榜、学期榜、赛事榜
- 增加后台批量解绑、换绑审批与导入模板下载
- 接入对象存储，保存轨迹附件或活动素材
- 后端按用户、运动、社交、运营等领域进一步拆分
