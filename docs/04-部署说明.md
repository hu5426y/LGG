# 校园乐跑部署说明

## 1. 部署目标

当前版本面向单校内部 MVP 上线，推荐部署形态：

- 后端与管理端使用容器镜像部署
- MySQL 使用云托管数据库
- Redis 使用云托管缓存
- 微信小程序使用正式 `AppID`

本地 `docker compose` 保留为开发环境，不作为生产部署方案。

## 2. 环境要求

开发环境：

- Docker / Docker Compose
- 微信开发者工具

生产环境：

- JDK 17+ 或容器运行环境
- MySQL 8
- Redis 7
- 微信小程序 `AppID / AppSecret`

## 3. 开发环境启动

在项目根目录执行：

```bash
chmod +x scripts/*.sh
./scripts/dev-up.sh
```

或直接：

```bash
docker compose up -d --build
```

默认开发服务：

- MySQL：`localhost:3307`
- Redis：`localhost:6380`
- Backend API：`http://127.0.0.1:8080`
- Admin Web：`http://127.0.0.1:5173`
- Health：`http://127.0.0.1:8080/actuator/health`
- Swagger：`http://127.0.0.1:8080/swagger-ui.html`

说明：

- 开发环境默认启用 `dev` profile
- 开发环境会执行 Flyway 迁移，并加载 `db/devdata` 下的开发种子数据
- 开发环境允许模拟跑步开关，仅用于联调与测试

## 4. 生产环境配置

生产环境建议使用环境变量管理配置。可从项目根目录的 [`.env.example`](../.env.example) 复制。

必须配置：

- `SPRING_PROFILES_ACTIVE=prod`
- `SPRING_DATASOURCE_URL`
- `SPRING_DATASOURCE_USERNAME`
- `SPRING_DATASOURCE_PASSWORD`
- `SPRING_DATA_REDIS_HOST`
- `SPRING_DATA_REDIS_PORT`
- `CAMPUSRUN_JWT_SECRET`
- `CAMPUSRUN_WECHAT_ENABLED=true`
- `CAMPUSRUN_WECHAT_APP_ID`
- `CAMPUSRUN_WECHAT_APP_SECRET`

建议配置：

- `CAMPUSRUN_SWAGGER_ENABLED=false`
- `CAMPUSRUN_BOOTSTRAP_ADMIN_ENABLED=true`
- `CAMPUSRUN_BOOTSTRAP_ADMIN_USERNAME`
- `CAMPUSRUN_BOOTSTRAP_ADMIN_PASSWORD`

生产环境行为：

- 不自动导入演示学生数据
- 不允许模拟跑步
- Swagger 默认关闭
- 微信登录未完成配置时会返回明确业务错误，而不是 `500`

## 5. 后端启动

使用容器时，将上述环境变量注入镜像即可。

如需单独启动后端：

```bash
cd backend
mvn spring-boot:run -Dspring-boot.run.profiles=prod
```

启动后可检查：

- `GET /actuator/health`
- `GET /actuator/info`

## 6. 管理端启动

开发环境：

```bash
cd admin-web
npm install
npm run dev
```

生产构建：

```bash
cd admin-web
npm install
npm run build
```

## 7. 微信小程序配置

1. 使用微信开发者工具导入 `miniapp/`
2. 在 `miniapp/config.js` 中配置正确的 API 地址
3. 配置微信后台合法请求域名
4. 确保后端已配置 `CAMPUSRUN_WECHAT_APP_ID` 与 `CAMPUSRUN_WECHAT_APP_SECRET`
5. 确保小程序已声明定位权限，并在真机或开发者工具中授权

说明：

- 当前版本默认走真实 GPS 路线采集
- 跑步结束会上报距离、时长、轨迹点集合
- 后端会校验最小时长、最小距离、轨迹点数量和异常速度

## 8. 学生名单导入

后台支持 `.csv` 和 `.xlsx` 导入。

推荐模板：

- [student-import-template.csv](./student-import-template.csv)

推荐列名：

- `student_no`
- `display_name`
- `college`
- `class_name`
- `username`
- `password`
- `status`

导入规则：

- 新学生必须提供初始密码
- 相同 `student_no` 会执行更新
- `status` 仅支持 `ACTIVE` 和 `DISABLED`
- 管理员可在后台禁用学生或重置微信绑定

## 9. 上线前检查

上线前至少完成以下核验：

1. 管理员账号初始化成功，且可登录后台
2. 学生名单已批量导入
3. 微信登录链路可完成“首次绑定 + 二次直登”
4. 真机跑步可完成“开始、暂停、继续、结束”
5. `/actuator/health` 返回正常
6. Swagger 在生产环境未对公网开放
7. JWT 密钥、数据库密码、微信密钥均通过环境变量注入
